## General Handoff Concepts

1. What is the primary purpose of handoffs in the Agents SDK?  
   a) To increase response creativity  
   b) To delegate tasks to specialized agents  
   c) To limit response length  
   d) To enforce tool usage  
   **Answer**: b) To delegate tasks to specialized agents  

2. How are handoffs represented to the LLM?  
   a) As instructions  
   b) As tools  
   c) As tokens  
   d) As prompts  
   **Answer**: b) As tools  

3. What is the default tool name for a handoff to an agent named "Refund Agent"?  
   a) refund_agent  
   b) transfer_to_refund_agent  
   c) handoff_refund_agent  
   d) call_refund_agent  
   **Answer**: b) transfer_to_refund_agent  

4. In which scenario are handoffs particularly useful?  
   a) When an agent needs to generate creative content  
   b) When different agents specialize in distinct tasks  
   c) When limiting token usage  
   d) When disabling tools  
   **Answer**: b) When different agents specialize in distinct tasks  

5. What happens when a triage agent hands off a query to a specialized agent?  
   a) The query is ignored  
   b) The specialized agent processes the query  
   c) The triage agent processes the query itself  
   d) The LLM stops execution  
   **Answer**: b) The specialized agent processes the query  

6. Which parameter in the Agent class is used to specify handoffs?  
   a) tools  
   b) handoffs  
   c) model_settings  
   d) instructions  
   **Answer**: b) handoffs  

7. What is the role of the `handoff_description` in an agent?  
   a) Defines the agent’s primary task  
   b) Provides a hint to the triage agent about when to hand off  
   c) Specifies the model’s temperature  
   d) Limits the input query length  
   **Answer**: b) Provides a hint to the triage agent about when to hand off  

8. What is the output of `result.last_agent` after a handoff?  
   a) The final response text  
   b) The name of the last agent that processed the query  
   c) The Agent class instance of the last agent  
   d) The triage agent’s instructions  
   **Answer**: c) The Agent class instance of the last agent  

9. How does the triage agent decide which agent to hand off to?  
   a) Based on the query’s token count  
   b) Based on the specialized agent’s name and instructions  
   c) Based on the model’s temperature  
   d) Based on random selection  
   **Answer**: b) Based on the specialized agent’s name and instructions  

10. What is the benefit of using handoffs in a customer support app?  
    a) Reduces response creativity  
    b) Allows specialized handling of tasks like refunds or FAQs  
    c) Limits the number of tools used  
    d) Increases token usage  
    **Answer**: b) Allows specialized handling of tasks like refunds or FAQs

## Creating Handoffs

11. How can a handoff be created in the Agents SDK?  
    a) By passing a tool to the model  
    b) By passing an Agent or Handoff object to the handoffs parameter  
    c) By setting the temperature parameter  
    d) By defining a new model  
    **Answer**: b) By passing an Agent or Handoff object to the handoffs parameter  

12. What does the `handoff()` function allow you to do?  
    a) Limit the response length  
    b) Customize the handoff process  
    c) Define the agent’s instructions  
    d) Disable tool usage  
    **Answer**: b) Customize the handoff process  

13. What is the default tool name generated by `Handoff.default_tool_name()`?  
    a) handoff_to_<agent_name>  
    b) transfer_to_<agent_name>  
    c) call_<agent_name>  
    d) agent_<agent_name>  
    **Answer**: b) transfer_to_<agent_name>  

14. Which function is used to create a customized handoff?  
    a) create_agent()  
    b) handoff()  
    c) run_sync()  
    d) model_settings()  
    **Answer**: b) handoff()  

15. What happens if no handoffs are specified in a triage agent?  
    a) The agent processes all queries itself  
    b) The agent throws an error  
    c) The agent uses default tools  
    d) The agent stops execution  
    **Answer**: a) The agent processes all queries itself  

16. What is the purpose of the `tool_name_override` in the `handoff()` function?  
    a) To override the agent’s instructions  
    b) To customize the tool name for the handoff  
    c) To limit the response tokens  
    d) To disable the handoff  
    **Answer**: b) To customize the tool name for the handoff  

17. Which parameter in the `handoff()` function overrides the default tool description?  
    a) tool_name_override  
    b) tool_description_override  
    c) input_filter  
    d) on_handoff  
    **Answer**: b) tool_description_override  

18. What is the role of the `instructions` parameter in a specialized agent?  
    a) Defines how the triage agent selects it  
    b) Specifies how the specialized agent processes queries  
    c) Limits the query length  
    d) Enables parallel tool calls  
    **Answer**: b) Specifies how the specialized agent processes queries  

19. How does the `handoff_description` differ from `instructions`?  
    a) It defines the agent’s task execution logic  
    b) It provides a hint for when to hand off to the agent  
    c) It limits the input query  
    d) It specifies the model’s settings  
    **Answer**: b) It provides a hint for when to hand off to the agent  

20. What is the effect of passing a normal agent directly to the `handoffs` parameter?  
    a) The handoff is disabled  
    b) The triage agent uses the agent’s name and instructions to decide handoff  
    c) The agent processes all queries itself  
    d) The agent uses a custom tool name  
    **Answer**: b) The triage agent uses the agent’s name and instructions to decide handoff

## Customizing Handoffs

21. What does the `is_enabled` parameter in the `handoff()` function control?  
    a) Whether the handoff is available to the LLM  
    b) The creativity of the response  
    c) The number of tokens in the response  
    d) The agent’s instructions  
    **Answer**: a) Whether the handoff is available to the LLM  

22. What is the default value of `is_enabled` in the `handoff()` function?  
    a) False  
    b) True  
    c) None  
    d) Auto  
    **Answer**: b) True  

23. If `is_enabled=False`, what happens to the handoff?  
    a) The handoff is hidden from the LLM  
    b) The handoff is always executed  
    c) The handoff uses default settings  
    d) The handoff throws an error  
    **Answer**: a) The handoff is hidden from the LLM  

24. What can the `is_enabled` parameter accept besides a boolean?  
    a) A string  
    b) A function that returns a boolean  
    c) A Pydantic model  
    d) A tool name  
    **Answer**: b) A function that returns a boolean  

25. What is the purpose of the `input_filter` parameter in the `handoff()` function?  
    a) To limit the response length  
    b) To modify or validate the input before it reaches the next agent  
    c) To override the tool name  
    d) To disable the handoff  
    **Answer**: b) To modify or validate the input before it reaches the next agent  

26. What does the `on_handoff` callback function do?  
    a) Validates the input query  
    b) Executes when the handoff is invoked  
    c) Defines the agent’s instructions  
    d) Limits token usage  
    **Answer**: b) Executes when the handoff is invoked  

27. What does the `input_type` parameter in the `handoff()` function specify?  
    a) The type of output expected from the agent  
    b) The expected structure of the input to the handoff  
    c) The tool name for the handoff  
    d) The temperature of the model  
    **Answer**: b) The expected structure of the input to the handoff  

28. How does `tool_description_override` affect the triage agent’s decision?  
    a) It limits the response length  
    b) It provides a specific hint for when to hand off  
    c) It defines the specialized agent’s task  
    d) It disables the handoff  
    **Answer**: b) It provides a specific hint for when to hand off  

29. What happens if `tool_name_override` is not specified in the `handoff()` function?  
    a) The handoff is disabled  
    b) The default tool name `transfer_to_<agent_name>` is used  
    c) The agent’s instructions are used as the tool name  
    d) An error is thrown  
    **Answer**: b) The default tool name `transfer_to_<agent_name>` is used  

30. Which parameter allows dynamic enabling/disabling of a handoff at runtime?  
    a) tool_name_override  
    b) tool_description_override  
    c) is_enabled  
    d) input_filter  
    **Answer**: c) is_enabled

## Differences Between Normal Agent and handoff()

51. What is the main difference between passing a normal agent and using the `handoff()` function?  
    a) Normal agent uses more tokens  
    b) handoff() provides more customization options  
    c) Normal agent disables handoffs  
    d) handoff() limits response length  
    **Answer**: b) handoff() provides more customization options  

52. How does a triage agent decide to hand off when using a normal agent?  
    a) Based on the query’s length  
    b) Based on the agent’s name and instructions  
    c) Based on the model’s temperature  
    d) Based on random selection  
    **Answer**: b) Based on the agent’s name and instructions  

53. When is the `handoff()` function preferred over passing a normal agent?  
    a) When simple handoff logic is needed  
    b) When precise control over handoff logic is required  
    c) When disabling handoffs  
    d) When limiting tokens  
    **Answer**: b) When precise control over handoff logic is required  

54. What does the `tool_description_override` in `handoff()` provide?  
    a) A generic description of the agent  
    b) A specific hint for when to hand off  
    c) The agent’s instructions  
    d) The response length limit  
    **Answer**: b) A specific hint for when to hand off  

55. How does a normal agent handoff differ in terms of specificity?  
    a) It is more specific than handoff()  
    b) It relies on generic agent instructions  
    c) It uses custom tool names  
    d) It disables customization  
    **Answer**: b) It relies on generic agent instructions  

56. What is a limitation of using a normal agent for handoffs?  
    a) It cannot be passed to the handoffs parameter  
    b) It lacks fine-tuned control over handoff logic  
    c) It increases token usage  
    d) It disables the triage agent  
    **Answer**: b) It lacks fine-tuned control over handoff logic  

57. What is an advantage of using the `handoff()` function?  
    a) It reduces response creativity  
    b) It allows specific scenarios for handoffs  
    c) It limits the number of agents  
    d) It disables input validation  
    **Answer**: b) It allows specific scenarios for handoffs  

58. How does `tool_name_override` affect the handoff process?  
    a) It changes the agent’s instructions  
    b) It customizes the tool name seen by the LLM  
    c) It limits the response length  
    d) It disables the handoff  
    **Answer**: b) It customizes the tool name seen by the LLM  

59. When might you avoid using the `handoff()` function?  
    a) When fine-tuned control is needed  
    b) When simple handoff logic is sufficient  
    c) When multiple agents are involved  
    d) When input validation is required  
    **Answer**: b) When simple handoff logic is sufficient  

60. What is the combined effect of `tool_name_override` and `tool_description_override`?  
    a) They limit the response tokens  
    b) They provide precise control over handoff identification and logic  
    c) They disable the handoff  
    d) They increase response creativity  
    **Answer**: b) They provide precise control over handoff identification and logic
